# 三层记忆架构模板

> 每本小说使用三层记忆系统管理上下文，确保长篇仿写时前后一致。

## 目录结构
```
novels/{novel-name}/
├── context/
│   ├── memory-outline.md      # 第一层：全书记忆大纲（≤2000字）
│   ├── recent-context.md      # 第二层：最近10章详细上下文（滚动窗口）
│   └── archives/              # 第三层：历史阶段归档
│       ├── stage-001-010.md
│       ├── stage-011-020.md
│       └── ...
```

---

## 第一层：memory-outline.md（全书记忆大纲）

**长度限制**：不超过2000字
**更新频率**：每10章重新生成一次
**用途**：写新章节时首先加载，获取全书大局观

```markdown
# 全书记忆大纲

## 故事进展摘要
（2-3句话概括从第1章到当前的整体剧情走向）

## 关键转折点
- 第X章：{转折事件一句话描述}
- 第Y章：{转折事件}
- ...

## 当前局势
（1-2段话描述当前各势力/角色的状态和关系格局）

## 核心角色状态
| 角色 | 当前状态 | 实力/地位 | 关键关系 |
|------|---------|----------|---------|
| {主角} | {当前处境} | {等级/身份} | {与谁的关系有变化} |
| {女主} | ... | ... | ... |
| ... | ... | ... | ... |

## 活跃伏笔
- 第X章埋设：{伏笔内容}，预计第Y章回收
- ...

## 已建立的世界规则
- {规则1：在故事中已确立的设定，后续不能违反}
- {规则2}
- ...
```

---

## 第二层：recent-context.md（最近10章详细上下文）

**长度限制**：无硬性限制，但通常不超过3000字
**更新频率**：每写完1章追加
**清理频率**：每10章归档最早部分
**用途**：写新章节时加载，获取最近发生的细节

```markdown
# 最近章节上下文

## 第{N}章
- **新增事实**：{本章确立的新信息}
- **角色状态变化**：{谁发生了什么变化}
- **新增冲突**：{本章新产生的矛盾}
- **解决的冲突**：{本章解决了什么问题}
- **使用的桥段**：{如：打脸、拍卖等}
- **章尾钩子**：{本章结尾的悬念是什么}

## 第{N+1}章
- ...

（保留最近10章的记录）
```

---

## 第三层：archives/stage-NNN-NNN.md（历史阶段归档）

**长度限制**：每个归档文件不超过500字
**更新频率**：每10章生成一个
**用途**：需要回查历史细节时按需加载，平时不加载

```markdown
# 第{start}-{end}章 阶段归档

## 阶段摘要
（3-5句话概括这10章的核心剧情发展）

## 本阶段关键事件
- 第{X}章：{事件}
- 第{Y}章：{事件}
- ...

## 角色变化
- {角色A}：从{状态1}变为{状态2}
- ...

## 伏笔记录
- 埋设：{伏笔内容}（第X章）
- 回收：{伏笔内容}（第Y章）
```

---

## 自动化更新逻辑

```
每写完1章 → 自动追加到 recent-context.md
    ↓
检查 current_chapter % 10 == 0？
    ├─ 否 → 继续下一章
    └─ 是 → 触发记忆压缩：
         1. recent-context.md 最早10章 → 压缩为 archives/stage-NNN-NNN.md
         2. 清理 recent-context.md 已归档部分
         3. 读取所有 archives/ + recent-context → 重建 memory-outline.md
```

## 写章节时的加载顺序

```
1. memory-outline.md      ← 必读（全书大局）
2. recent-context.md      ← 必读（近期细节）
3. archives/*.md          ← 按需读取（只在遇到跨度大的回忆/伏笔时）
```
