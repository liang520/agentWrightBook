# 原作分析模板

> 分析一部原作时，按此模板在 sources/{source-name}/ 下创建文件。

## 目录结构
```
sources/{source-name}/
├── meta.json           # 基本信息
├── analysis.md         # 整体分析（填写本模板）
└── chapters/           # 原作章节内容
    ├── 001.md
    ├── 002.md
    └── ...
```

## meta.json 格式
```json
{
  "title": "原作标题",
  "author": "原作者",
  "genre": "都市/修仙/甜宠/...",
  "total_chapters": 200,
  "source_url": "来源链接",
  "word_count": 0,
  "status": "analyzing | analyzed | ready",
  "analysis_progress": 0
}
```

## analysis.md 内容要求

### 一、基本信息
- 题材类型
- 总章数
- 核心卖点一句话总结

### 二、故事骨架
- 开局（第1-N章）：怎么开的，主角初始状态
- 发展线（按阶段划分）：每个阶段的核心冲突
- 高潮点：列出所有重大转折的章节号
- 结局：怎么收的

### 三、角色功能分析
列出所有重要角色，标注：
- 角色名
- 角色原型（对应 library/archetypes 中的哪种）
- 首次出场章节
- 核心功能（推动剧情/制造冲突/感情线/搞笑...）
- 退场章节（如果有）

### 四、章节功能标注
对每一章标注其"功能"，格式：
```
第001章：【开局引入】主角出场，交代背景，展示低谷状态
第002章：【冲突触发】被退婚/被欺负/获得金手指
第003章：【首次爽点】第一次打脸/展示实力
...
```

### 四（扩展）、短篇改编压缩标注（`/short analyze` 时使用）

在基础标注之后，追加以下字段：

```
第001章：【开局引入】主角出场，交代背景，展示低谷状态
- 章节功能: 主线推进
- 核心事件: 主角灵魂寄居冰箱，发现杀人秘密
- 可压缩度: 不可压缩
- 依赖关系: 被第2/3/4章依赖（主角身份设定）
- 关键角色: 楚荣、王香、张贤
- 伏笔: 埋设（冰箱中的神秘能量）
```

字段说明：
- **章节功能**：`主线推进` | `支线展开` | `人物塑造` | `世界观铺垫` | `过渡/灌水`
- **可压缩度**：`不可压缩`（核心情节）| `可合并`（可与相邻章合并）| `可删减`（灌水/过渡）
- **依赖关系**：标注哪些后续章节依赖本章内容（伏笔、角色登场、设定引用等）
- **伏笔**：`埋设（内容）` | `回收（内容）` | `无`

### 五、节奏分析
- 平均每章字数
- 爽点密度（几章一个小爽点）
- 高潮间隔（几章一个大高潮）
- 章尾钩子类型统计
