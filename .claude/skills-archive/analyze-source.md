# 分析原作小说

## 触发条件
用户提供了一部原作小说的内容（文件或文本），需要进行结构化分析。

## 交互规则
**所有需要用户选择或确认的环节，必须以编号选择题形式呈现，用户只需回复数字即可。**

## 执行步骤

### 1. 确认素材
- 扫描 `sources/` 下所有子目录，列出已有的原作
- 如果有章节内容但尚未分析的原作，以选择题展示：
```
请选择要分析的原作：
1. 斗破苍穹短篇（200章，未分析）
2. 都市之王（150章，未分析）
3. ➕ 导入新的原作
```
- 如果选择导入新原作，询问：
```
原作章节内容在哪里？
1. 已放在 sources/ 目录下（告诉我目录名）
2. 我手动粘贴内容
3. 从网页采集（需要提供网址）
```

### 2. 创建 meta.json
读取 `docs/source-template.md` 中的 meta.json 格式，创建基本信息文件。

确认题材类型：
```
这部原作属于哪个题材？
1. 都市
2. 玄幻/修仙
3. 甜宠/言情
4. 校园
5. 悬疑
6. 其他（请说明）
```

### 3. 逐章分析
对每一章标注功能，格式：
```
第001章：【功能标签】一句话概括本章干了什么
```

功能标签包括但不限于：
- 开局引入、背景交代、角色引入、冲突触发
- 打脸爽点、实力展示、升级突破、获得机缘
- 感情推进、误会制造、真相揭露、伏笔埋设
- 高潮对决、阶段收尾、新篇开启

### 4. 角色分析
列出所有重要角色，对照 `library/archetypes/` 中的原型进行分类，记录：
- 角色名、原型、首次出场章节、核心功能、退场章节

分析完成后展示角色列表，让用户确认：
```
以下是识别出的主要角色，请确认：
1. ✅ 确认无误
2. ❌ 有遗漏，我补充
3. 🔄 某个角色的原型分类不对，我修改
```

### 5. 节奏分析
统计：
- 每章平均字数
- 爽点密度
- 高潮间隔
- 章尾钩子类型

### 6. 输出
将分析结果写入 `sources/{source-name}/analysis.md`，格式参照 `docs/source-template.md`。
更新 `sources/{source-name}/meta.json` 的 status 为 "ready"。

完成后提示：
```
分析完成！接下来你想：
1. 基于这部原作创建新小说（/new-novel）
2. 查看分析报告详情
3. 分析另一部原作
```

## 注意事项
- 分析过程中每完成10章暂停汇报一次进度
- 如果章节数量超过50章，分批分析，每批20章
