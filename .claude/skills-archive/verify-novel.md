# 全书校验

## 触发条件
- 整本书仿写完成后（meta.json 的 current_chapter = total_chapters）
- 用户手动调用想在中途做一次全面检查

## 交互规则
**所有需要用户选择或确认的环节，必须以编号选择题形式呈现。**

## 执行步骤

### 1. 选择小说
```
请选择要校验的小说：
1. 都市逆袭王（200/200章，已完成）
2. 校园甜宠记（150/150章，已完成）
```

### 2. 七维度全面校验

#### 2.1 映射完整性检查
- 读取 `config/character-map.md` 中所有原作角色名
- 读取 `config/setting-map.md` 中所有原作设定名
- 逐章扫描 `chapters/*.md`，检查是否有原作名词泄漏
- 输出：泄漏位置列表（章节号 + 行号 + 原文）

#### 2.2 角色弧线检查
- 读取 `context/memory-outline.md` 和所有 `archives/*.md`
- 对每个主要角色，检查其成长/变化是否连贯
- 检查项：
  - 角色性格是否前后一致（除非有合理的成长理由）
  - 角色实力是否符合升级曲线（不能突然变强无理由）
  - 角色关系的变化是否有铺垫
- 输出：角色弧线报告

#### 2.3 伏笔闭合检查
- 读取 `config/foreshadowing.md`
- 检查所有伏笔的状态
- 标记未回收的伏笔为问题
- 输出：伏笔状态表（已回收/未回收/过期）

#### 2.4 逻辑一致性检查
- 读取 `context/memory-outline.md` 中的"已建立的世界规则"
- 抽查10-15章，检查是否有违反世界规则的情节
- 检查时间线是否合理（事件先后顺序）
- 输出：逻辑问题列表

#### 2.5 节奏检查
- 读取原作 `sources/{name}/analysis.md` 中的章节功能分布
- 对比新书各章的功能分布
- 检查：
  - 爽点是否分布均匀（不能连续10章没有爽点）
  - 高潮节奏是否与原作匹配
  - 是否有功能缺失（原作有的功能新书没覆盖）
- 输出：节奏对比分析

#### 2.6 风格漂移检查
- 读取 `config/style.md` 中的风格定义和系统提示词
- 随机抽取5-10章（均匀分布在全书各阶段）
- 检查语言风格是否与定义一致
- 标记风格偏移的章节
- 输出：风格一致性报告

#### 2.7 字数统计
- 统计每章字数
- 计算平均值、中位数、最大值、最小值
- 标记异常（过短<800字或过长>5000字的章节）
- 输出：字数分布统计

### 3. 输出校验报告
将完整报告写入 `novels/{novel-name}/verification-report.md`：

```markdown
# 全书校验报告：{title}

## 总览
- 总章数：{N}
- 总字数：{N}
- 校验时间：{date}

## 评分
| 维度 | 评分 | 说明 |
|------|------|------|
| 映射完整性 | ✅/⚠️/❌ | {说明} |
| 角色弧线 | ✅/⚠️/❌ | {说明} |
| 伏笔闭合 | ✅/⚠️/❌ | {说明} |
| 逻辑一致性 | ✅/⚠️/❌ | {说明} |
| 节奏匹配 | ✅/⚠️/❌ | {说明} |
| 风格一致 | ✅/⚠️/❌ | {说明} |
| 字数规范 | ✅/⚠️/❌ | {说明} |

## 问题详情
### 严重问题（必须修复）
- ...

### 注意事项（建议修复）
- ...

### 改进建议
- ...
```

### 4. 处理问题
```
校验完成！发现 {N} 个问题。
1. 🔧 自动修复所有严重问题
2. 🔧 自动修复全部问题
3. 📋 逐个确认后修复
4. 📝 仅记录，暂不修复
5. 📖 查看完整报告
```

### 5. 修复后重新校验
如果执行了修复，自动重跑受影响的校验维度，确认问题已解决。

### 6. 最终确认
```
✅ 全书校验通过！

1. 📊 查看完整校验报告
2. 📖 查看全书统计
3. 🚀 准备发布
4. 🔙 返回主菜单（/novel-status）
```
