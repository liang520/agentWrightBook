# 创建新小说

## 触发条件
用户想要基于某个已分析的原作（sources/中status为ready的）创建一本新的仿写小说。

## 交互规则
**所有需要用户选择或确认的环节，必须以编号选择题形式呈现，用户只需回复数字即可。绝不使用填空题或开放式提问。如果需要用户提供名称等自定义内容，也要先给出AI生成的候选项让用户选择，最后附加"N. 自定义（请输入）"选项。**

## 执行步骤

### 1. 选择原作
扫描 `sources/` 下所有 status 为 "ready" 的原作，展示选择题：
```
请选择要仿写的原作：
1. 斗破苍穹短篇（都市，200章）
2. 都市之王（都市，150章）
3. ...
```

### 2. 选择世界观
扫描 `library/worlds/` 下所有模板，展示选择题：
```
请选择新书的世界观：
1. 修仙体系（练气→筑基→金丹...）
2. 都市异能（觉醒者→D级→C级...）
3. 都市豪门（底层→小康→豪门...）
4. 校园（普通学生→年级名人...）
5. 自定义（请描述）
```

### 3. 选择语言风格
扫描 `library/styles/` 下所有模板，展示选择题：
```
请选择语言风格：
1. 幽默吐槽 — "你知道我是谁吗？""不知道，但我知道你很快就会是个躺着的人。"
2. 冷酷硬派 — "最后一次机会。""你威胁我？""不，通知你。"
3. 热血燃向 — "我从来不需要胜算。我只需要一个不认输的理由！"
4. 甜宠温馨 — "你为什么对我这么好？""反正不是因为喜欢你。""那你脸红什么？"
5. 自定义（请描述）
```

### 4. 新书名称
基于选定的原作、世界观和风格，AI 生成5个候选书名：
```
请选择新书名称：
1. {AI生成名1}
2. {AI生成名2}
3. {AI生成名3}
4. {AI生成名4}
5. {AI生成名5}
6. 自定义（请输入书名）
```

### 5. 创建角色映射
读取原作 `analysis.md` 中的角色列表，对每个角色依次操作：

**5a. 选择角色原型**
```
原作角色「张三」（主角，隐忍复仇型）
请选择新角色的原型：
1. 隐忍复仇型 — 表面低调隐忍，有仇必报
2. 霸道强势型 — 自信碾压，不屑解释
3. 扮猪吃虎型 — 表面废物，实际深藏不露
4. 热血少年型 — 正义感强，永不服输
5. 腹黑算计型 — 表面温文尔雅，步步为营
6. 保持和原作相同的原型
```

**5b. 选择新角色名**（AI根据世界观生成候选名）
```
请选择「张三」的新名字：
1. 李明
2. 陈峰
3. 周逸
4. 林昊
5. 自定义（请输入）
```

**5c. 选择说话方式**
```
「李明」的说话方式：
1. 简洁冷酷 — "滚" "不配"
2. 随和低调 — "我只是运气好" "别误会"
3. 幽默吐槽 — "你确定？我可以给你个重新选择的机会"
4. 霸道直接 — "我说了算" "不接受反驳"
5. 自定义（请描述）
```

对女主、主要反派、重要配角重复以上流程。次要角色由AI自动映射，展示结果让用户确认：
```
以下次要角色已自动映射，请确认：
| 原作 | 新角色 | 原型 |
|------|--------|------|
| 王五 | 赵磊   | 忠犬兄弟型 |
| 刘六 | 钱浩   | 嚣张纨绔型 |

1. ✅ 全部确认
2. 🔄 我要修改某个（告诉我编号）
```

### 6. 确认设定映射表
AI 根据原作分析和选定世界观自动生成设定映射表，展示给用户：
```
以下是自动生成的设定映射，请确认：

| 原作 | 新设定 |
|------|--------|
| 灵石 | 能量晶石 |
| 筑基期 | D级觉醒者 |
| 天剑宗 | 天盾局 |
| ... | ... |

1. ✅ 全部确认
2. 🔄 我要修改某项（告诉我哪个）
3. ➕ 有遗漏，我补充
```

### 7. 最终确认
展示完整配置摘要：
```
📖 新书配置摘要
──────────────
书名：{title}
原作：{source}
世界观：{world}
风格：{style}
角色数量：主角1 + 女主1 + 反派3 + 配角5
设定映射：{N}条

1. ✅ 确认，创建所有文件
2. 🔄 返回修改某个配置
```

### 8. 创建文件
确认后自动创建所有文件：
- `novels/{novel-name}/meta.json`
- `novels/{novel-name}/config/character-map.md`
- `novels/{novel-name}/config/setting-map.md`
- `novels/{novel-name}/config/style.md`（从 library/styles/ 复制对应风格，含系统提示词）
- `novels/{novel-name}/config/foreshadowing.md`（空表）
- `novels/{novel-name}/context/memory-outline.md`（空的全书记忆大纲）
- `novels/{novel-name}/context/recent-context.md`（空的最近上下文）
- `novels/{novel-name}/context/archives/`（空目录）
- `novels/{novel-name}/chapters/` 目录

完成后提示：
```
新书创建完成！接下来：
1. 开始写第1章（/write-chapter）
2. 查看完整配置
3. 返回修改配置
```

## 文件格式
参照 `docs/novel-template.md` 中的所有格式定义。
