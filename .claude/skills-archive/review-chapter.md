# 审校章节

## 触发条件
用户想要审校某本小说的某个章节或某个范围的章节。

## 交互规则
**所有需要用户选择或确认的环节，必须以编号选择题形式呈现，用户只需回复数字即可。**

## 执行步骤

### 1. 选择小说
如果有多本小说，展示选择题：
```
请选择要审校的小说：
1. 都市逆袭王（已写45章）
2. 校园甜宠记（已写12章）
```

### 2. 选择审校范围
```
请选择审校范围：
1. 最新一章（第45章）
2. 最近5章（第41-45章）
3. 最近10章（第36-45章）
4. 全部章节（第1-45章）
5. 自定义范围（输入起止章节）
```

### 3. 加载配置
读取该小说的所有配置文件：
- character-map.md
- setting-map.md
- style.md
- foreshadowing.md
- context.md

### 4. 逐章检查
对每一章执行以下检查：

#### 映射一致性检查
- 扫描全文，是否出现了原作的角色名（不应该出现）
- 扫描全文，是否出现了原作的设定名词（不应该出现）
- 角色的性格和说话方式是否与 character-map.md 一致

#### 前后一致性检查
- 对照 context.md，是否有事实矛盾
- 角色在不该出场的时候出场了吗
- 时间线是否合理

#### 伏笔检查
- foreshadowing.md 中标记为"待回收"的伏笔，是否到了该回收的时候
- 本章是否埋了新伏笔但没记录

#### 风格检查
- 语言风格是否与 style.md 一致
- 是否有出戏的现代梗或不合时宜的表达

#### 质量检查
- 章尾是否有钩子
- 是否有明显的逻辑漏洞
- 对话是否自然

### 5. 输出审校报告
```markdown
## 审校报告：{novel-name} 第{start}-{end}章

### 问题汇总
- 🔴 严重（{N}个）
- 🟡 注意（{N}个）
- 🟢 建议（{N}个）

### 详细问题列表
1. 🔴 第{X}章第{Y}行：出现原作角色名「张三」，应为「李明」
2. 🟡 第{X}章：角色「苏晴」的说话方式偏离设定（设定为毒舌，实际太温柔）
3. 🟢 第{X}章：章尾钩子力度不够，建议加强悬念
...

### 通过项
- ✅ 设定映射一致性
- ✅ 时间线逻辑
- ...
```

### 6. 处理问题
```
发现{N}个问题，如何处理？
1. 🔧 自动修复所有严重问题
2. 🔧 自动修复全部问题
3. 📋 逐个确认后修复
4. 📝 仅记录，暂不修复
```

如果选择逐个确认，对每个问题展示：
```
问题：第{X}章第{Y}行出现原作角色名「张三」
建议修复：替换为「李明」

1. ✅ 接受修复
2. ❌ 跳过
3. ✏️ 用其他方式修复（请说明）
```

### 7. 完成
```
审校完成！
- 已修复：{N}个问题
- 跳过：{N}个问题
- 剩余待处理：{N}个

接下来：
1. 继续写作（/write-chapter）
2. 再审校一次
3. 查看进度（/novel-status）
```
