# 批量仿写

## 触发条件
用户想要一次性连续仿写多个章节。

## 交互规则
**所有需要用户选择或确认的环节，必须以编号选择题形式呈现，用户只需回复数字即可。**

## 执行步骤

### 1. 选择小说
如果有多本小说，展示选择题：
```
请选择要批量写作的小说：
1. 都市逆袭王（当前第45章）
2. 校园甜宠记（当前第12章）
```

### 2. 选择批量数量
```
要连续写几章？
1. 5章（第46-50章）
2. 10章（第46-55章）
3. 20章（第46-65章）
4. 写到完结（还剩{N}章）
5. 自定义数量
```

### 3. 选择检查频率
```
每写几章暂停检查一次？
1. 每5章检查一次（推荐）
2. 每3章检查一次（谨慎）
3. 每10章检查一次（快速）
4. 不检查，全部写完再看
```

### 4. 确认开始
```
即将开始批量写作：
- 小说：{title}
- 范围：第{start}章 → 第{end}章
- 检查频率：每{N}章暂停
- 记忆压缩：每10章自动执行

1. ✅ 开始
2. 🔄 修改设置
```

### 5. 循环写作
对每一章执行 `write-chapter` 的核心流程：

```
每写1章自动执行：
  ├─ 加载 style.md（含系统提示词）+ memory-outline.md + recent-context.md + 映射表
  ├─ 读取原作对应章节 + 功能标注
  ├─ 生成新章节（遵守风格人格、严格映射替换）
  ├─ 自查质量（映射、一致性、风格、钩子）
  ├─ 保存章节文件
  ├─ 更新 recent-context.md（追加本章事实）
  ├─ 更新 foreshadowing.md（如有）
  ├─ 更新 meta.json（current_chapter +1）
  │
  └─ 检查：current_chapter % 10 == 0？
       ├─ 否 → 继续下一章
       └─ 是 → 自动执行记忆压缩：
            ├─ 归档最早10章到 archives/
            ├─ 清理 recent-context.md
            ├─ 重建 memory-outline.md
            └─ 汇报压缩结果
```

### 6. 检查点
每到用户设定的检查频率时暂停，展示：
```
📊 已完成第{start}-{current}章

| 章节 | 功能 | 字数 |
|------|------|------|
| 第46章 | 打脸爽点 | 2100字 |
| 第47章 | 感情推进 | 1800字 |
| ... | ... | ... |

📦 记忆状态：
- 记忆大纲：{X}字
- 当前记忆窗口：第{A}-{B}章
- 已归档阶段：{N}个

⚠️ 发现的问题：{如有}

1. ✅ 继续写下一批
2. ⏸️ 暂停，我要检查
3. 🔄 回退重写第{N}章
4. ⏹️ 停止批量写作
```

### 7. 新角色/新设定处理
批量写作中遇到新角色/新设定时，暂停并展示选择题（同 write-chapter 的步骤8），处理完后自动继续。

### 8. 完成汇报
```
✅ 批量写作完成
──────────────
小说：{title}
本次写作：第{start}章 - 第{end}章
累计进度：{current}/{total} ({百分比}%)
新增角色：{如有}
新增伏笔：{如有}
记忆压缩：执行了{N}次
总字数：{X}字

接下来：
1. 审校本批章节（/review-chapter）
2. 继续批量写作（/batch-write）
3. 查看进度（/novel-status）
4. 全书校验（/verify-novel）— 仅当全书完成时
```
