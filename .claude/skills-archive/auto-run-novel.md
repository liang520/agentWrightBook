# 全自动整本书运行

## 触发条件
用户想要一键式完成一本小说的完整仿写流程。

## 交互规则
**配置阶段（分析+新建）以选择题形式与用户交互。写作阶段全自动运行，每10章暂停汇报一次。**

## 完整流程

### 阶段一：分析原作（自动调用 /analyze-source 流程）

1. 扫描 `sources/` 目录，展示选择题：
```
请选择要仿写的原作：
1. {已有原作1}（{N}章，{status}）
2. {已有原作2}
3. ➕ 导入新的原作
```

2. 如果原作尚未分析（status != ready），自动执行分析：
   - 确认题材（选择题）
   - 逐章标注功能（每10章汇报）
   - 提取角色并归类（展示让用户确认）
   - 节奏分析
   - 生成 analysis.md

3. 分析完成后自动进入阶段二。

### 阶段二：配置新书（自动调用 /new-novel 流程）

全选择题交互：
1. 选世界观 → 2. 选风格 → 3. 选书名 → 4. 角色映射 → 5. 设定映射 → 6. 最终确认

创建完所有配置文件（含三层记忆目录结构）后自动进入阶段三。

### 阶段三：全书仿写（核心自动化循环）

```
确认开始：
即将开始全书自动仿写：
- 小说：{title}
- 总章数：{N}章
- 记忆压缩：每10章自动执行
- 审校检查：每10章自动执行
- 暂停汇报：每10章暂停一次

1. ✅ 开始全自动写作
2. 🔄 修改设置
```

#### 自动循环逻辑

```
对每一章（从第1章到第total_chapters章）：
  │
  ├─ 加载上下文：
  │   style.md（系统提示词）
  │   memory-outline.md + recent-context.md
  │   character-map.md + setting-map.md + foreshadowing.md
  │
  ├─ 读取原作对应章节 + 功能标注
  │
  ├─ 生成新章节（遵守风格人格、映射替换、上下文一致）
  │
  ├─ 自查质量（5项检查）
  │
  ├─ 保存章节 → 更新 recent-context.md → 更新 foreshadowing.md → 更新 meta.json
  │
  ├─ 遇到新角色/设定 → 暂停，选择题处理后继续
  │
  ├─ current_chapter % 10 == 0？
  │   └─ 是 → 执行：
  │       1. 记忆压缩（/compress-memory）
  │       2. 审校最近10章（/review-chapter 的检查逻辑，自动修复严重问题）
  │       3. 暂停汇报：
  │          ┌──────────────────────────────────────┐
  │          │ 📊 阶段完成：第{start}-{end}章          │
  │          │ 进度：{current}/{total} ({百分比}%)     │
  │          │ 本阶段字数：{X}字                       │
  │          │ 累计字数：{Y}字                         │
  │          │ 新增角色：{如有}                         │
  │          │ 新增伏笔：{如有}                         │
  │          │ 审校问题：严重{N}个，已自动修复           │
  │          │ 记忆大纲：{Z}字                         │
  │          │                                        │
  │          │ 1. ✅ 继续写下一阶段                     │
  │          │ 2. ⏸️ 暂停，我要检查                     │
  │          │ 3. 🔄 回退重写某章                       │
  │          │ 4. ⏹️ 停止（进度已保存，随时可恢复）      │
  │          └──────────────────────────────────────┘
  │
  └─ 继续下一章
```

### 阶段四：全书校验（自动调用 /verify-novel 流程）

全书写完后自动执行七维度校验：
1. 映射完整性
2. 角色弧线
3. 伏笔闭合
4. 逻辑一致性
5. 节奏匹配
6. 风格一致
7. 字数统计

输出 verification-report.md，展示结果：
```
✅ 全书仿写+校验完成！

📖 {title}
总章数：{N}章
总字数：{X}字
校验结果：{通过/有问题}

1. 📊 查看校验报告
2. 🔧 修复校验问题
3. 📖 查看全书统计
4. 🚀 准备发布到百度作家平台
5. 🔙 返回主菜单
```

## 中断恢复机制

**任何时候中断（会话断开、用户暂停、系统故障），都可以无缝恢复。**

恢复逻辑：
1. 用户再次执行 `/auto-run-novel`
2. 系统检测到 `novels/{name}/meta.json` 中 status = "writing" 且 current_chapter > 0
3. 展示：
```
🔄 检测到未完成的小说：
{title} — 已写到第{current_chapter}/{total_chapters}章

1. ▶️ 从第{current_chapter+1}章继续
2. 🔄 回退到第{N}章重新开始
3. 🗑️ 放弃这本，重新开始
```

恢复时自动加载：
- memory-outline.md（全书大纲）
- recent-context.md（最近10章）
- 所有 config 文件

**无需人工重建上下文，三层记忆系统保证恢复后上下文完整。**

## 与其他 Skills 的关系

```
/auto-run-novel 是总控，内部调用：
  ├─ /analyze-source  （阶段一）
  ├─ /new-novel        （阶段二）
  ├─ /write-chapter    （阶段三，每章核心逻辑）
  ├─ /compress-memory  （阶段三，每10章）
  ├─ /review-chapter   （阶段三，每10章审校逻辑）
  └─ /verify-novel     （阶段四）
```

用户也可以单独调用任何子 Skill，不一定非要通过 `/auto-run-novel`。
