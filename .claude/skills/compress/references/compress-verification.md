# Phase 4：终验（12 维度）

## 前置条件

meta.json `status` 为 `"verifying"`

## 12 维度检查

| # | 维度 | 说明 |
|---|------|------|
| 1 | 总字数 | target × 0.95 ≤ 实际 ≤ target（硬上限不超标） |
| 2 | 章节完整性 | 所有新章文件存在，无缺漏 |
| 3 | 伏笔完整性 | 重映射后伏笔表 vs 压缩稿，逐条确认 |
| 4 | 因果链完整性 | 跨章因果链两端都在压缩稿中保留 |
| 5 | 情绪曲线 | 高潮/转折章节情绪力度未减弱 |
| 6 | 原作覆盖率 | 所有非"删除"原作章的核心事件在压缩稿中有体现 |
| 7 | 风格一致性 | 抽检 5 章，对照 style.md |
| 8 | A/B 对照 | 随机 3-5 章，原稿 vs 压缩稿对比 |
| 9 | 原作名子串扫描 | 如原书有 character-map.md，扫描原作角色名子串是否泄漏 |
| 10 | 跨章状态一致性 | 提取【】系统提示中的状态词，按事件名分组检查矛盾 |
| 11 | 数值链闭合 | 提取货币/积分系统数值，验证跨章余额闭合（无数值系统则 N/A） |
| 12 | 伏笔回收质量评估 | 对已回收伏笔按3项打分（仅Suggestion，不阻断） |

**不需要的维度**（与 `/short` 的区别）：
- 映射泄漏扫描（不换角色/设定，没有泄漏问题）
- 等级体系一致性（不改设定体系）

**维度 1：总字数**
```
统计全部压缩后章节的总字数（统一口径）
通过条件：target_total × 0.95 ≤ 总字数 ≤ target_total
```

**维度 2：章节完整性**
```
检查 novels/{name}-compressed/chapters/ 下：
  - 所有章节文件 001.md 到 {compressed_chapters}.md 存在
  - 标题格式正确
  - 编号连续无遗漏
```

**维度 3：伏笔完整性**
```
读取 foreshadowing.md（重映射版）
逐条在压缩后的章节中搜索确认：
  - 每个伏笔的埋设在对应新章中仍有体现
  - 每个伏笔的回收在对应新章中仍有体现
标记任何遗漏
```

**维度 4：因果链完整性**
```
从 compress-plan.md 事件清单中提取跨章因果链：
  - 章 A 的事件 X → 导致章 B 的事件 Y
逐条验证因果链两端是否都在压缩稿中保留
```

**维度 5：情绪曲线**
```
从 compress-plan.md 的事件清单中提取情绪高点章节
逐一阅读这些章节的压缩稿：
  - 情绪力度是否因压缩而减弱？
  - 是否存在"铺垫被删导致高潮来得突兀"的问题？
  - 连续章节之间的情绪过渡是否自然？
```

**维度 6：原作覆盖率**
```
遍历映射表中所有非"删除"行：
  对每行的"核心保留事件"：
    在对应新章的压缩稿中确认存在
标记覆盖率不足的章节
```

**维度 7：风格一致性**
```
抽取 5 章（均匀分布），与原书 style.md 对照
检查：语言风格、叙述视角、用词倾向是否一致
```

**维度 8：A/B 对照抽检**
```
随机抽取 3-5 章（覆盖不同压缩策略）
将原稿和压缩稿对比：
  - 压缩稿是否丢失了让人印象深刻的细节？
  - 合并章的起承转合是否自然？
  - 压缩稿是否保持了原稿的节奏感？
```

**维度 9：原作名子串扫描**
```
如果原书有 character-map.md（即原书是 /novel 或 /short 产出）：
  - 对 character-map.md 左列每个原作角色名，提取≥2字的名字部分（去姓氏）
  - 用该子串扫描全部压缩后的章节
  - 白名单过滤：跳过该子串作为常见词的用法（如"依然"作为副词、"清"作为形容词前缀）
  - 判断方法：如果该子串出现在对话中作为称呼、或紧跟在姓氏后面 → 标记为泄漏
  - 如果该子串是常见汉语词汇且用法符合语境 → 跳过
如果原书无 character-map.md → 跳过此维度，标记 N/A
```

**维度 10：跨章状态一致性**
```
提取压缩稿中所有【】系统提示中的状态词（如【已觉醒】【已突破】等）
按事件名/角色名分组，检查跨章状态是否存在矛盾
（如角色在第5章【已觉醒】但第8章又出现【未觉醒】）
标记矛盾项
```

**维度 11：数值链闭合**
```
提取压缩稿中涉及货币/积分/等级数值的描述
验证跨章余额是否闭合（收入 - 支出 = 当前余额）
如果全书无数值系统 → 标记 N/A
```

**维度 12：伏笔回收质量评估**（仅 Suggestion，不阻断）
```
对 foreshadowing.md 中标记"已回收"的伏笔，读取回收章节相关段落
按3项打分（每项1分）：
  ① 是否明确点名回收该伏笔（1分）
  ② 是否推动后续情节（1分）
  ③ 是否带来情绪/关系变化（1分）
3分=A级（优秀），2分=B级（合格），0-1分=C级（不足）
C级 → Suggestion："伏笔回收力度不足，建议扩写"
```
