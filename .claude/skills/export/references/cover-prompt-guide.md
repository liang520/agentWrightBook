# 封面提示词生成标准

## 核心原则

### 1. 理解受众张力
封面必须击中目标读者的审美偏好。不同频道/题材的读者要的东西完全不同：

| 受众 | 要的视觉张力 | 封面应表现 |
|------|------------|-----------|
| 女频甜宠 | 被保护感、反差萌、危险中的温柔 | 男主护女主的姿态、男主的身体细节（伤疤=经历、手=温柔）、两人之间的距离感和张力 |
| 女频虐恋 | 纠结、隐忍、欲言又止 | 背对背或侧脸、一方伸手另一方未接、冷色调中一抹暖 |
| 女频重生/逆袭 | 蜕变感、掌控感 | 女主居中、眼神冷静坚定、前世/今生的视觉对比 |
| 男频玄幻 | 力量感、压迫感、世界观宏大 | 男主俯视/仰视视角、巨大的背景建筑/生物、能量特效 |
| 男频都市 | 低调的强、反差身份 | 普通外表暗藏锋芒的暗示（如西装袖口露出的伤疤） |
| 悬疑/灵异 | 不安感、窥探感 | 局部遮挡、异常光影、不对称构图 |

**必须先判断频道和题材（Step 3.7 的投稿分类结果），再决定封面的视觉策略。**

### 2. 表现力标准
封面提示词必须做到让 AI 绘图模型"看到"完整画面，不能含糊：

**必须描述的 5 个层次**：

```
【人物细节】— 不是"a handsome man"，而是：
  - 具体五官特征（从原文提取：眉骨高度、瞳色、疤痕位置、肤色）
  - 具体身体细节（从原文提取：伤疤、缺损、体型、发型）
  - 具体穿着（从原文提取：材质、颜色、湿/干、破损程度）
  - 具体姿态（从原文提取：正在做什么动作、手放在哪里、视线朝向）
  - 具体表情（从原文提取：不是"angry"，而是"dead calm, almost bored"）

【人物关系】— 两人之间的空间语言：
  - 谁主动谁被动（谁在伸手、谁在接）
  - 距离感（近到呼吸交错 / 隔着车窗 / 背对背）
  - 对比（瓷器与铁管、精致与粗粝、干净与带血）

【场景还原】— 从原文中提取最有画面感的那一幕：
  - 不选"男女主站在一起"的静态合影
  - 选故事中最有张力的一个"定格瞬间"（如：拉开车门的那一秒）
  - 场景中的环境细节（地面积水、蒸汽、远处霓虹、背景倒地的人）

【2.5D 深度层次】— 必须明确标注前中后景：
  foreground: 最近的视觉元素（雨滴、飘落的花瓣、模糊的前景物体）
  mid-ground: 主体人物，清晰焦点
  background: 环境氛围，适度虚化

【色彩情绪】— 不是随便配色，而是服务于情绪：
  - 主色调匹配故事基调（暴雨夜=深蓝+琥珀车灯）
  - 点缀色来自角色标志物（女主的酒红礼服、男主灰色帽衫）
  - 光源必须有物理来源（路灯、车灯、窗户），不是漫无目的的打光
```

## 执行流程

```
1. 确定受众（从 Step 3.7 投稿分类结果读取频道+题材）
2. 选"定格瞬间"：
   - 通读前5章，找到最有视觉冲击力+情感张力的一幕
   - 优先选：两人初次相遇/关键转折/最能体现书名主题的场景
3. 回原文抓细节：
   - 读取该场景所在章节的完整文本
   - 提取：人物外貌描写、动作描写、环境描写、对话中暗示的表情
   - 特别关注：手的细节、眼神描写、身体距离、光线/天气
4. 按 5 层结构组织提示词（人物细节→人物关系→场景还原→深度层次→色彩情绪）
5. 末尾附加技术参数：--ar 2:3 --v 6.1 --style raw
```

## 禁止事项

- 禁止用泛泛描述替代具体细节（"a handsome man with scars" → 必须写清哪只手、什么疤、怎么来的）
- 禁止脱离原文想象（封面画面必须能在原文中找到对应段落）
- 禁止忽略受众差异（女频封面不能用男频的构图逻辑）
- 禁止省略深度层次（没有前中后景分层的提示词一律打回重写）
- 禁止使用 AI 绘图常见的空洞形容词（beautiful, gorgeous, stunning → 用具体的五官/光影描述替代）

## 输出格式示例

提示词按以下结构组织，每个层次用【】标注：

```
2.5D [风格] illustration, [频道] web novel cover,
[场景类型] scene:

【中景主体·角色A】[具体外貌+穿着+姿态+表情，从原文提取]
【中景主体·角色B】[具体外貌+穿着+姿态+表情，从原文提取]（如有）
【环境】[场景细节，从原文提取]
【2.5D 深度层次】foreground: ... / mid-ground: ... / background: ...
【色彩】[主色调+点缀色+光源]
【风格】[2.5D具体类型 + 受众审美定位]

--ar 2:3 --v 6.1 --style raw
```
