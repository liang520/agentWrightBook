# 原作导入流程

## 全局规则

1. **全程自动** — 导入过程不等待用户输入
2. **字数统计统一口径** — 遵循 CLAUDE.md "字数统计规范"（去空格、去 Markdown 标记后的字符数，含标点）
3. **编码安全** — 自动检测并转码，绝不产生乱码

## 用法

```
/import {文件路径}
/import /path/to/novel.txt
/import /path/to/novel.epub
```

## 核心流程

### Step 1：接收文件路径并验证

```
1. 从用户消息中提取文件路径参数
2. 验证文件是否存在（使用 Bash 工具 ls 或 Read 工具）
3. 验证文件格式：
   - .txt → 纯文本流程
   - .epub → EPUB 流程
   - 其他格式 → 提示"暂不支持该格式，目前支持 txt 和 epub"，终止
```

### Step 2：编码检测与转码

**仅 txt 格式需要**（epub 内部为 UTF-8/XML）

```
1. 使用 file 命令检测文件编码：
   file -I {文件路径}

2. 如果检测结果不明确，读取文件前 1000 字节，尝试以下编码依次解码：
   - UTF-8（优先）
   - GB18030（兼容 GBK/GB2312）
   - BIG5（繁体中文）
   - UTF-16

3. 选择不产生乱码的编码进行转码：
   iconv -f {检测到的编码} -t UTF-8 {文件路径} > {临时文件}

4. 如果文件已经是 UTF-8，跳过转码
```

### Step 3：EPUB 解析（仅 epub 格式）

```
1. 解压 epub 文件（epub 本质是 zip）：
   unzip -o {文件路径} -d {临时目录}

2. 解析 content.opf 或 META-INF/container.xml 找到内容文件列表

3. 按 spine 顺序读取各 xhtml/html 章节文件

4. 去除 HTML 标签，提取纯文本内容

5. 将所有章节文本合并为一个连续文本流（后续步骤统一处理章节拆分）
```

### Step 4：章节分隔符检测与拆分

```
1. 自动检测章节分隔符模式（按优先级尝试）：

   a. 中文数字模式："第[零一二三四五六七八九十百千万]+章"
      正则：第[零一二三四五六七八九十百千万]+章

   b. 阿拉伯数字模式："第\d+章"
      正则：第\d+章

   c. 英文模式："Chapter \d+"
      正则：Chapter\s+\d+

   d. 数字编号模式：行首独立数字（如 "1"、"2"...），仅在前三种都未匹配时尝试

   e. 如果所有模式均未检测到（<3 个匹配）：
      → 提示"未检测到标准章节分隔符"
      → 展示文本前 2000 字符
      → 请用户提供自定义正则表达式
      → 用用户提供的正则重新拆分

2. 按检测到的分隔符拆分章节：
   - 每个匹配位置作为新章节的起始
   - 分隔符所在行作为章节标题
   - 分隔符之前的内容（如有）作为"前言"或"楔子"，编号为 000.md

3. 提取章节标题：
   - 从分隔符行中提取完整标题（如"第一章 天降奇缘"→ 标题为"天降奇缘"）
   - 如果分隔符行只有章节号无标题，标题留空
```

### Step 5：生成章节文件

```
1. 自动生成 slug（目录名）：
   - 尝试从文件名提取书名
   - 如果书名为中文：Claude 根据中文推断拼音首字母缩写（如"斗破苍穹"→ "dpcq"），不依赖外部拼音库
   - 如果书名为英文：使用小写连字符格式（如"My Novel"→ "my-novel"）
   - 如果无法提取：使用文件名（去扩展名，转小写，空格转连字符）
   - 检查 sources/ 下是否已存在同名目录，如有则追加数字后缀（-2, -3...）

2. 创建目录结构：
   sources/{slug}/
   ├── chapters/
   └── meta.json

3. 逐章写入文件：
   - 文件名格式：三位数零填充（001.md, 002.md, ..., NNN.md）
   - 如果有前言/楔子：保存为 000.md（不计入 total_chapters，下游 Skill 的章节校验从 001.md 起始）
   - 每章第一行为：# {章节标题}
     - 如果原始标题为"第一章 天降奇缘"→ 写入 `# 第一章 天降奇缘`
     - 如果原始标题为"第一章"（无标题）→ 写入 `# 第一章`
   - 章节正文从标题下一行开始
   - 去除正文中的多余空行（连续3个以上空行压缩为2个）
```

### Step 6：统计与生成 meta.json

```
1. 统计总字数（统一口径）：
   遍历所有章节文件 → 去除 Markdown 标记 → 去除空白字符 → 计算字符数

2. 统计章节数

3. 尝试从文本中提取书名和作者名：
   - 检查文件首部（前 500 字符）是否有书名/作者信息
   - 检查文件名是否包含书名信息
   - 如果无法提取，书名使用 slug，作者留空

4. 生成 meta.json：
{
  "title": "书名",
  "author": "作者（如有）",
  "slug": "slug",
  "status": "raw",
  "total_chapters": N,
  "word_count": NNNNN,
  "encoding_detected": "UTF-8/GB18030/...",
  "chapter_pattern": "检测到的分隔符模式",
  "imported_from": "原始文件路径",
  "imported_at": "YYYY-MM-DD"
}
```

### Step 7：汇报导入结果

```
═══ 原作导入完成 ═══
文件：{原始文件路径}
书名：{title}
作者：{author}
编码：{encoding} → UTF-8
章节数：{total_chapters}
总字数：{word_count}（统一口径）
存储路径：sources/{slug}/
状态：raw（待分析）

下一步：
1. /novel analyze — 分析原作结构（用于等长仿写）
2. /short analyze — 分析原作结构（用于短篇改编）
═══════════════════
```

## 错误处理

```
- 文件不存在 → 提示"文件不存在：{路径}"，终止
- 编码无法识别 → 提示"无法识别文件编码，请确认文件是否为文本文件"，终止
- 章节拆分失败（0 章） → 提示"未检测到章节分隔符"，请求用户提供正则
- epub 解析失败 → 提示"EPUB 文件解析失败，请检查文件是否完整"，终止
- slug 冲突 → 自动追加数字后缀，不终止
```

## 注意事项

1. 导入只负责拆分和存储，不做任何分析（分析由 `/novel analyze` 或 `/short analyze` 完成）
2. meta.json 的 status 为 "raw"，表示尚未分析
3. 如果用户提供的是已拆分好的目录（sources/{name}/chapters/ 已有文件），提示"该原作已存在，如需重新导入请先删除 sources/{slug}/ 目录"
