# Phase 7.5：独立交叉审查

## 触发条件
Phase 7 完成后（无论是否有问题）

## 流程

```
1. 启动2个独立审查 Agent（subagent_type: code-reviewer，并行）

   Agent A（逻辑审查）：
   - 因果链完整性（跨章事件是否有因有果）
   - 反派行为合理性（已做决策+已知信息的该做未做）
   - 数值一致性（魂魄数、分身数、时间线等具体数字跨章比对）
   - 设定规则执行（如"认出即死"诅咒是否全书一致）
   - 叙事完整性（重大反转是否有前文暗示和落地交代）

   Agent B（连贯性 + 可读性审查）：
   - 上下文衔接（逐对相邻章检查状态连续性）
   - 人名一致性（角色名全书统一+配置↔正文一致）
   - 原作痕迹（全量禁词扫描+标志短语扫描）
   - 新设定铺垫（首次使用的能力/概念是否有前文铺垫）
   - 【编辑视角追加项】以网文编辑身份审查：
     - 全书节奏是否有明显拖沓或过赶的段落
     - 角色对话是否自然（标记机械感/书面感过强的对话）
     - 是否有明显的 AI 写作痕迹（重复句式、模板化描写）
     - 编辑视角发现的问题分级为 Suggestion，仅"读者完全看不懂"升为 Warning

2. 独立性要求
   - 两个 Agent 不读 verification-report.md（避免锚定效应）
   - 必须逐章通读全部章节（不是抽查）
   - 各自独立输出结构化问题清单

3. 主 Agent 汇总
   - 与 Phase 7 终验报告交叉比对
   - 逐一验证真伪（读章节原文确认）
   - 三方一致确认 → 直接修复
   - 两方发现一方未发现 → 验证后决定
   - 仅一方发现 → 验证真伪，可能为误报

4. 修复后复审
   - 再启动2个 Agent 聚焦修复点 + 修复是否引入新问题
   - 通过 → 进入修复阶段收尾（由主 SKILL.md 决定具体 Phase 编号）
   - 不通过 → 再修一轮（最多2轮总计）

5. 收敛门禁
   - Critical 未解决 → 不通过
   - Warning 超过5个 → 不通过
   - 最大迭代2轮
   - 2轮后仍有 Critical → 输出残留清单，标注"人工决定"
```

## meta.json 状态
- 进入时：status → "cross-reviewing"
- 通过后：由 SKILL.md 主流程决定下一步（素材回流 → complete，或修复阶段：short=Phase 8，novel=Phase 7）
