# Phase 7：全书校验（23 维度）

**前置条件**：meta.json current_chapter == total_chapters 且总字数 ≥ 45000

**触发前**：如果 recent-context.md 有未归档章节，先执行 Phase 6

更新 meta.json status → "verifying"

## 23 维度检查

### 1. 映射泄漏扫描（最关键）

详细扫描逻辑见 [../../shared/leak-scan.md](../../shared/leak-scan.md)

### 2. 章节完整性
- 所有章节文件存在（001.md 到 {total}.md）
- 标题格式正确（`# 第N章 标题`）
- 编号连续无遗漏

### 3. 字数分布（统计口径见 CLAUDE.md）
- 统计每章字数
- 标记异常短（<800字）或异常长（>2500字）的章节
- 与 compression-map.md 的目标字数对比偏差

### 4. 等级体系一致性
- 搜索 setting-map.md 中已映射的等级术语原作词是否残留
- 扩展：术语变体统一性检查
  - 从 setting-map.md 提取映射的等级/设定术语
  - 检查同一术语在全书中是否存在不同写法（如"练气X层"vs"练气X阶"）
  - 如存在混用 → Warning

### 5. 角色名+称呼一致性
- 主要角色名全书统一（不出现不同称呼混用）
- **称呼用词一致性**：对核心角色，提取全书中对其的称呼方式，检查同一说话者在同一语境类型（正式/非正式）中对同一人的称呼是否前后一致。无故变化=Suggestion（不是Warning）。豁免：场合正式度不同、刻意隐瞒关系、情绪激动等，且该原因在上下文中可推断
- 扩展：对 Top 8 核心角色，检查以下属性跨章一致性：
  - 恒定属性（年龄差、出身、性别）：不允许变化，冲突=Warning
  - 可演化属性（等级、状态、计数）：只允许合理的单向推进（升级、受伤→痊愈），不允许无故回退
  - 方法：提取属性首次出现的描述，与后续对比。无故回退=Warning

### 6. 伏笔闭合
- 伏笔对账：逐条读取 foreshadowing.md，在章节中搜索确认
- 标记脏数据（表中有但章节中无）
- 清除脏数据后标记未回收伏笔

### 7. 风格一致性
- 抽查 max(10章, 全书15%) 章，上限20章。均匀分布抽样，确保覆盖开头、中段、高潮、结尾各部分，与 style.md 对照

### 8. 记忆档案完整性
- 所有 stage 归档文件存在且覆盖全部章节

### 9. 审校发现汇总
- 整合所有后台审校 Agent 的结果

### 10. 非关键建议
- 记录可改进但非必须的问题

### 11. 压缩质量总评（短篇专属）
- 总字数是否达标（≥ 45000）
- 与 compression-map.md 的字数目标偏差
- 情节因果链完整性
- 节奏曲线评估
- **压缩跳跃检测**：
  - 对 compression-map 中"合并"策略的章节：
    读取上一章末段摘要和本章首段引入
    检查是否出现未铺垫的新角色/地点/事件
    检查上章钩子是否在本章有呼应
    出现断层 → Warning

### 12. 跨章状态一致性
- 提取全部章节中【】包裹的系统提示文本
- 筛选含"完成/未完成/触发/失败/达成/进行中"等状态词
- 按事件名分组，检查同一事件是否出现矛盾状态
- 排除合理的状态推进（"触发"→"进行中"→"完成"）

### 13. 数值链闭合
- 从 setting-map.md 提取货币/积分系统名称
- 搜索全部含该货币名的行
- 验证跨章余额闭合
- 允许"隐性扣除"但必须有文本交代

### 14. 情绪曲线
- 从 memory-outline.md 提取关键转折点章节
- 逐一阅读评估情绪力度是否充足
- 检查连续章节的情绪过渡是否自然

### 15. 因果链完整性
- 提取跨章因果链（章A事件X → 导致章B事件Y）
- 逐条验证因果链两端都在成稿中保留
- **后果合理性子项**：对高严重度事件额外检查——
  - L3事件（犯罪/重伤/死亡）：5章内必须有后果交代
  - L2事件（公开丑闻/冲突）：3章内必须有后果交代
  - 豁免：如有权势压制/私了/超凡能力等文本解释 → 不报 Warning
  - 后果明显轻于事件严重程度且无解释 → Warning
- **承诺/威胁追踪（轻量版）**：
  - 搜索对话中将来时承诺（"一定会""必须""我发誓""等着瞧""上奏""挑战""我知道""告诉你""下落""秘密""真相""答应你""保证""不会放过""等我回来""记住"）
  - 检查后续5章内是否有回应
  - 完全无回应 → Warning（"对话承诺悬置"），仅标记不阻断

### 16. 设定规则执行一致性（检索预筛 + Agent 裁决）
- 从 setting-map.md + config/style.md + 正文前10章中提取已建立的世界观规则声明
- 每条规则生成关键词，用 Grep 搜索全书含该关键词的段落
- 筛出疑似冲突片段（规则被违反但无解释的场景）
- 仅把疑似片段交给 Agent 裁决是否真正冲突
- 确认冲突 → Warning

### 17. 伏笔回收质量评估（仅 Suggestion，不阻断）
- 对 foreshadowing.md 中标记"已回收"的伏笔，读取回收章节相关段落
- 按3项打分（每项1分）：
  ① 是否明确点名回收该伏笔（1分）
  ② 是否推动后续情节（1分）
  ③ 是否带来情绪/关系变化（1分）
- 3分=A级（优秀），2分=B级（合格），0-1分=C级（不足）
- C级 → Suggestion："伏笔回收力度不足，建议扩写"

### 18. 姓氏+通称泄漏扫描
- 已包含在映射泄漏扫描的 shared/leak-scan.md 第4项中
- 此维度确认第4项扫描结果已执行

### 19. 时间线连续性
硬规则：
- 读取 context/timeline.md（如存在），检查倒计时是否单调递减
- 如无 timeline.md，从正文提取时间标记
软规则：
- 跨章时间跳跃超过2天且无文本交代 → Warning
- 同一倒计时在不同章节出现不同数值 → Critical
章内时间一致性检查：
- 对每章提取所有时间标记（早上/下午/晚上/深夜/今天/明天/昨天/周X等）
- 检查同一章同一场景内的时间标记是否自洽
- 如"下午"场景中出现"明晚" → Warning（章内时间矛盾）

### 20. 承诺/威胁追踪（全局）
- 与维度15的轻量版互补，此处做全书级扫描
- 汇总维度15发现的所有"对话承诺悬置"Warning，统计全书悬置率
- 悬置率 > 30% → Warning（"全书承诺回应率偏低"）

### 21. 跨章+章内重复事件检测
- 提取每章核心事件（3-5个），标准化为：事件类型 + 参与者 + 行为 + 结果
- 重点检查并行写作批次边界章节（如ch7/ch8, ch14/ch15等）±1章窗口
- 发现相同事件在不同章节被完整描述两次 → Warning
- 如果是"回忆/复盘"语境中的重提 → 降级为Suggestion
- 章内重复检测：
  对每章内部，检查是否有高度相似的段落（相同描写、相同对话、相同场景）出现两次
  章内重复 → Warning

### 22. 视角一致性
- 从 style.md 或 meta.json 确定本书的主视角角色
- 扫描全书，检查是否有段落从其他角色的内心视角描写
  （非主视角角色的心理活动、主观感受、内心独白）
- 排除：对话中转述、神识远观的间接获取、全知旁白交代事实
- 不一致 → Warning
- 注意：某些章节（如前传/回忆）可能合理切换视角，需结合上下文判断

### 23. 时空自洽性（距离-时间交叉验证）
- 提取全书中涉及地理移动的描写（从A地到B地），记录每次出现的距离/时间设定
- 检查同一路线在不同章节中的距离/时间描述是否一致（如"快马三天"不能在另一章变成"次日到达"）
- 检查兵力调动、追击、赶赴等行动的时间窗口是否与已建立的距离设定匹配
- 注意：本维度不检查章内时间标记自洽（已由维度19覆盖），只检查距离-时间交叉矛盾
- 不一致=Warning
