# Prompt 模板

## 强制约束

组装写作 prompt 时，**必须**按此模板填充，**禁止**手动压缩或重写系统提示词。

## 模板

```
=== SYSTEM ===
{读取 config/style.md "## 系统提示词" 到文件末尾的全部内容，原文粘贴，不做任何修改}

=== USER ===
【字数硬要求 — 第一优先级】
本章目标字数：{TARGET}字。允许范围：{TARGET*0.9}-{TARGET*1.1}字。
你必须写到至少{TARGET*0.9}字。不满足字数的输出将被拒绝重写。

【名词替换清单 — 第二优先级】
绝对不能出现左侧任何词：
{从 character-map.md 和 setting-map.md 提取完整替换清单}

{IF 有上次泄漏反馈 — 此区块在重试时必须放在 === USER === 最开头，字数要求之前}
❌ 上次输出包含以下泄漏，必须彻底杜绝：
{泄漏词列表 + 替换方案}
{ENDIF}

【写作任务】《{TITLE}》第{CH}章
- 对应原作：第{SRC_CHAPTERS}章
- 压缩策略：{STRATEGY}
- 叙事功能：{FUNCTION}
- 核心事件：{EVENTS}

{IF 弧间边界章节}
【弧间过渡要求】
本章是新弧的开篇。前一弧的主角是{PREV_ARC_CHAR}，本弧切换到{CUR_ARC_CHAR}。
- 开头用1-2句自然过渡（场景转换、时间推移、或旁白引入）
- 不要硬切——读者需要知道故事视角在移动
{ENDIF}

## 全书记忆
{读取 context/memory-outline.md 全文}

## 最近章节上下文
{读取 context/recent-context.md 最近3章的摘要}

## 前章内容
{IF N > 1: 读取 chapters/{N-1}.md 最后500字原文粘贴}
{IF N == 1: 写"本章为全书第1章，无前章内容。"}

## 事件边界
- 前章终止事件：{PREV_END_EVENT}
- 本章起始事件：{START_EVENT}
- 本章终止事件：{END_EVENT}
- 后章起始事件：{NEXT_START_EVENT}

## 原作参考情节
{读取原作章节，提取核心情节要点，5-8条}

## 事实锁定
{从 details-lock.md 提取与本章相关的条目}

## 角色说话方式
{从 character-map.md 提取本章涉及角色的说话方式}

【防重叠规则 - 强制执行】
1. 前章内容已给出。从前章结尾处自然接续，绝不重复前章事件。
2. 只负责本章事件 ID 列表中的事件，不得越界。
3. 写到本章终止事件落地即止。不写余波。
4. 不编造 details-lock 中未列的具体数字/物理细节。

【输出要求】
- 以"# 第{CH}章"开头，标题自拟
- 章末必须有悬念钩子
- 不要输出任何JSON或元数据
- 必须写到至少{TARGET*0.9}字
```

## 填充规则

1. **系统提示词**：`cat config/style.md | sed -n '/^## 系统提示词$/,$ p' | tail -n +2` — 原文，不改一个字
2. **前章正文**：`tail -c 1500 chapters/{N-1}.md` — 取末尾约500字，不是手写摘要
3. **替换清单**：从 character-map + setting-map 的左列提取，包括别名列
4. **弧间边界**：如果 compression-map 中本章与前章的弧线标签不同，触发弧间过渡
5. **字数**：从 compression-map 读取目标字数，计算 90% 和 110% 范围
