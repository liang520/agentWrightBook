# 映射泄漏扫描（通用）

适用于 /novel 和 /short 的终验阶段。/condense 和 /compress 不改编角色设定，不需要此维度。

## 扫描流程

```
1. 全名匹配（含别名）
   - 从 character-map.md 左列提取每一个原作角色名
   - 如果存在"原作别名"列，同时提取每个角色的所有别名（以"、"分隔）
   - 从 setting-map.md 左列提取每一个原作设定词
   - 同时读取 character-map.md 中的"龙套列表"区块（如有），提取龙套原作名一并扫描
   - 用 Grep 逐词扫描全部章节（角色名+别名+龙套名+设定词）
   - 称谓方式扫描：除正式名和别名外，还要扫描角色的称谓模式
     如"{姓}某"（裴某→凌某）、"{原作名}——不，{新名}"（叶——不，苏晚）、"{原作称号}城主"等
     称谓模式从 character-map.md 的"原作别名"列中提取（格式支持通配符如"X城主"）
   - 向后兼容：如果 character-map.md 没有"原作别名"列，跳过别名扫描和称谓方式扫描
   - 输出：泄漏位置列表（文件 + 行号 + 原文）
   - 执行确认：扫描完成后输出统计——"共扫描{N}个角色名+{M}个别名+{K}个设定词，命中{H}处"
   - 如果 K=0（setting-map 左列未被扫描），标记为执行错误并重新扫描

2. 合并角色检查
   - 对 character-map.md 中标记为"合并"的角色，提取被合并方的名称
   - 在全部章节中搜索被合并方名称，确认其不以独立角色形态出现
   - 如出现 → 标记为 Critical

3. 子串扫描
   - 对 character-map.md 左列每个原作角色名，提取≥2字的名字部分（去姓氏）
   - 用该子串扫描全部章节
   - 白名单过滤：跳过该子串作为常见词的用法
     （如"依然"作为副词、"清"作为形容词前缀）
   - 判断方法：
     → 如果该子串出现在对话中作为称呼、或紧跟在姓氏后面 → 标记为泄漏
     → 如果该子串是常见汉语词汇且用法符合语境 → 跳过

4. 姓氏+通称组合扫描
   - 对 character-map.md 左列每个原作角色名，提取姓氏（第一个字）
   - 扫描以下组合：{姓}同学、{姓}姐、{姓}哥、{姓}老师、{姓}先生、{姓}小姐、{姓}叔、{姓}阿姨
   - 白名单：如果姓氏为极常见字（王/李/张/陈/刘/赵/黄/周/吴/郑）且出现在明显非称呼语境中 → 跳过
   - 判断方法：如果"{姓}+通称"在对话中作为称呼使用 → 标记为 Critical
```
